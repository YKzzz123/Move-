Move! V2 - 治愈系身心协同 Web 应用初步技术方案 (v2.0)
一、 项目愿景
本项目是一个融合了“西方星座番茄钟”与“东方中医经络、古典文学”理念的现代 Web 健康应用。旨在通过边缘计算视觉检测用户的微运动，配合“气”的累积与大语言模型的个性化文本反馈，为久坐人群提供兼具实用性与人文关怀的身心疗愈体验。
二、 系统架构设计
本项目采用标准的前后端分离架构，确保系统的高效性与数据安全。
前端框架 (Client-side): Vue 3 + Vite + CSS 库 (如 Tailwind CSS)。负责 UI 渲染、番茄钟逻辑及与用户的直接交互。
边缘视觉引擎 (Edge AI): Google MediaPipe Tasks API for Web (JavaScript SDK)。核心： 所有的摄像头流获取、Pose 关键点提取及动作准确度计算均在浏览器端本地完成，实现真正的低延迟与零视频流外泄。
后端框架 (Server-side): FastAPI (Python)。负责业务逻辑流转、数据校验、以及作为调用大语言模型的安全代理层。
数据库引擎 (Database): MySQL。作为核心数据持久化中心，存储用户档案、运动流水、日记及应用内的高价值内容。
生成式 AI 模型: Gemini API。负责基于用户的运动表现和当前状态，生成带有古典文学/疗愈色彩的个性化鼓励文案。
三、 核心模块与实现路径
1. 本地动作检测逻辑 (基于 Web 端)
利用前端的 MediaPipe 提取人体关键点坐标，通过几何规则进行动作判定：
动作设计与文化融合： 微运动(眼部、手部、肩颈)的设计将与传统中医经络穴位概念深度结合。例如，手部动作引导可结合“合谷穴”的舒展，肩颈动作结合“大椎”的放松，在技术实现上对应特定关节的角度和高度阈值。
完成度判定： 当计算出的动作准确度 > 70% 时，前端判定为动作完成，并向后端发送结算请求。
2. 激励体系与前端表现 (简化表现，增强人文)
为深化项目的人文主义色彩，并在 Demo 阶段保持开发的高效性，对视觉表现进行了针对性设计：
“气”值积分系统： 摒弃传统的金币概念，用户完成运动后收获“气”。“气”值作为核心驱动力，用于同步小猫的状态以及解锁后续的潜在内容。
轻量化视觉呈现： 虚拟猫咪暂不采用复杂的 Live2D 物理演算，而是采用 12 种星座对应的 2D 静态贴图序列（Sprite Animation）。每只小猫保留三种核心状态交互：待机态（修行中）、开心态（得气/任务完成）、失落态（力竭/任务未完成）。通过 Vue 的过渡动画实现平滑切换。
3. 数据层与高价值内容管理
通过 MySQL 管理以下核心数据结构：
Users (用户表): 记录用户的基本信息、生日（用于映射星座小猫）、当前的形态以及累计的“气”值。
Sessions (运动流水表): 记录每一次番茄钟的起止时间、动作类型以及是否达标，为未来的数据图表化提供支撑。
Diaries (日记本): 允许用户记录每日的心情与感悟，提供私密的情感沉淀空间。
Energy Station (能量站): 存储应用生成的古典文学/疗愈金句，以及用户主动收藏的心仪文案。
四、 系统核心业务流程 (Workflow)
初始化与设置： 用户打开网页，注册/登录并输入生日。系统根据生日分配对应的十二星座初始猫咪角色。
久坐监测与待机： 用户开启番茄钟检测模块，小猫进入预设的“待机态”。用户可调整计时规则（时长、分段），应用进入倒计时。
触发提醒与决策： 计时结束，弹出运动提醒窗口。用户需决策是“开始训练”还是“忽略提醒”。
边缘验证环节： 若选择训练，进入微运动选择界面。前端调用摄像头，基于 MediaPipe 进行实时动作检测与验证。
结算与状态同步：
动作达成（>70%）：前端发送成功信号给 FastAPI 后端，后端更新 MySQL 中的“气”值积分与流水。
前端接收更新响应后，将猫咪状态切换为“开心态”。
FastAPI 异步调用 Gemini API，结合中医/古典文化生成一句针对本次运动的疗愈性反馈，推送到前端展示。
内容沉淀： 用户可随时进入“能量站”，将刚才收到的金句收藏至 MySQL 数据库，或在日记模块写下当前感受。
4. 系统静态资源
1）对应十二星座的12钟小猫形态（AI生成的2D形象），每一只猫都有3个初始互动（待机态，任务完成的开心态，任务未完成态的失落态）
2）三套微运动动作（眼部、手部、肩颈）。动作由AI设计（固定）
